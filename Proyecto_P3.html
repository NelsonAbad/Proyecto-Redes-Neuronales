3. Construccin del conjunto de datos y preprocesamiento

En esta seccin se genera el conjunto de datos a partir de las carpetas locales del proyecto.
El dataset contiene imgenes de dgitos manuscritos capturados por estudiantes, distribuidos en:

  • Train/: usado para entrenar el modelo
  • Test/: usado nicamente para evaluar el desempeo final

Cada carpeta contiene subcarpetas del 0 al 9, donde cada imagen pertenece a su clase correspondiente.
Para facilitar el entrenamiento se crea un data pipeline con image_dataset_from_directory.

3.1 Creacin de los datasets

Se generan tres conjuntos:

  • train_ds 70% del conjunto Train, usado para entrenamiento
  • val_ds 30% restante, usado para validacin
  • test_ds imgenes completamente nuevas (nunca vistas durante el entrenamiento)

Todas las imgenes se cargan como escala de grises y se redimensionan a 2828 px para mantener compatibilidad con arquitecturas tipo MNIST.

3.2 Normalizacin

Las imgenes se normalizan al rango ([0,1]) dividiendo entre 255.
Adems, se aplica una inversin de color (1.0 - image) porque los dgitos del dataset son:

  • oscuros sobre fondo claro, pero el modelo funcionaba mejor aprendiendo nmeros claros sobre fondo oscuro.

Esto mejora la estabilidad del entrenamiento.

3.3 Aumento de datos (Data Augmentation)

Para mejorar la generalizacin del modelo y simular variaciones reales en la escritura, se aplican transformaciones aleatorias nicamente al conjunto de entrenamiento:

  • Rotaciones pequeas (10%)
  • Zoom aleatorio
  • Traslaciones horizontales y verticales

Estas operaciones ayudan al modelo a volverse ms robusto ante nmeros mal alineados o ligeramente deformados.

3.4 Optimizacin del pipeline

Finalmente, los datasets pasan por prefetch(AUTOTUNE) para permitir que TensorFlow prepare el siguiente batch mientras el modelo entrena, acelerando considerablemente el proceso.

Este pipeline deja listo un flujo eficiente y completamente preprocesado de imgenes para construir y entrenar los diferentes modelos de CNN.

5. Definicin de la arquitectura CNN base

En esta seccin se construye el modelo convolucional principal que servir como referencia para comparar con las dems arquitecturas alternativas. El modelo se implementa con tf.keras.Sequential, y est compuesto por las siguientes capas:

5.1 Capas convolucionales

  1. Conv2D (32 filtros, 33, ReLU, padding='same')

    • Extrae patrones locales como bordes o trazos de los dgitos.
    • El padding same conserva el tamao espacial de la imagen.

  2. MaxPooling2D (22)

    • Reduce a la mitad las dimensiones del mapa de caractersticas.
    • Ayuda a disminuir el ruido y evitar el sobreajuste.

  3. Conv2D (64 filtros, 33, ReLU)

    • Extrae caractersticas ms complejas, como curvas o cambios de grosor.

  4. MaxPooling2D (22)

    • Contina reduciendo la dimensionalidad y conservando la informacin relevante.

  5. Conv2D (128 filtros, 33, ReLU)

    • ltima capa convolucional, enfocada en capturar patrones de alto nivel propios de cada dgito.

5.2 Capas densas (fully connected)

  • Flatten
    Convierte los mapas de caractersticas en un vector unidimensional para la etapa de clasificacin.

  • Dense(64, ReLU)
    Acta como capa intermedia para aprender combinaciones no lineales de las caractersticas extradas.

  • Dropout(0.3)
    Apaga aleatoriamente el 30% de las neuronas durante el entrenamiento.
    Su propsito es reducir el overfitting y mejorar la capacidad de generalizacin del modelo.

  • Dense(10, softmax)
    Capa final de clasificacin.
    Produce un vector de 10 probabilidades correspondientes a cada dgito (09).

5.3 Compilacin

El modelo se compila con:

  • Optimizador: Adam, utilizado por su eficiencia y estabilidad.
  • Funcin de prdida: categorical_crossentropy, adecuada para clasificacin multiclase.
  • Mtrica: accuracy, para evaluar precisin durante entrenamiento y validacin.

La arquitectura utilizada en este proyecto est compuesta por tres bloques convolucionales seguidos de capas densas al final. El modelo inicia con una capa Conv2D de 32 filtros con un tamao de kernel de 33 y activacin ReLU. Esta capa es responsable de detectar patrones simples en la imagen, como bordes y trazos iniciales. Posteriormente, se aplica una capa de MaxPooling2D, la cual reduce a la mitad la resolucin espacial (de 2828 a 1414), conservando solo la informacin ms relevante de cada regin.

El segundo bloque convolucional utiliza 64 filtros de 33 y tambin est acompaado por una capa de max pooling. En este punto, los mapas de caractersticas representan estructuras ms complejas, como intersecciones y curvas gruesas que conforman los nmeros. El tercer bloque aplica 128 filtros y ya no utiliza pooling; aqu la red aprende representaciones an ms abstractas, como partes completas del dgito. Tras este proceso de extraccin de caractersticas, una capa Flatten transforma los mapas de caractersticas tridimensionales en un vector de tamao 6272.

A continuacin, se agrega una capa densa de 64 neuronas, que acta como un clasificador interno capaz de relacionar las caractersticas extradas en los niveles anteriores. Debido a que esta capa contiene ms de 400,000 parmetros, constituye la parte ms pesada del modelo y la que concentra la mayor capacidad de aprendizaje. Para reducir el sobreajuste un riesgo comn considerando que el dataset del saln contiene imgenes ruidosas y poco uniformes se incluye una capa Dropout con tasa de 0.3. Finalmente, se utiliza una capa Dense con 10 salidas y activacin softmax, correspondiente a las 10 clases de dgitos (09).

Con esta arquitectura, la informacin visual atraviesa diferentes niveles de abstraccin: desde bordes simples hasta representaciones completas de cada nmero, permitiendo al modelo aprender a clasificar dgitos escritos a mano de forma eficiente.

6. Entrenamiento del modelo con Early Stopping

En esta etapa se entrena la arquitectura convolucional definida previamente utilizando los conjuntos de entrenamiento y validacin. Para garantizar que el modelo no se sobreentrene, se emplea la tcnica de Early Stopping, la cual permite detener el entrenamiento automticamente cuando el rendimiento ya no mejora.

6.1 Configuracin de Early Stopping

Se utiliza la clase EarlyStopping incluida en Keras, configurada de la siguiente forma:

monitor = 'val_accuracy' El criterio monitoreado es la exactitud obtenida en el conjunto de validacin. Esto permite identificar el punto donde el modelo deja de generalizar correctamente.

patience = 5 Si la exactitud de validacin no mejora durante 5 pocas consecutivas, el entrenamiento se detiene. Este valor previene seguir entrenando cuando el modelo empieza a sobreajustarse.

restore_best_weights = True Asegura que, al finalizar el entrenamiento, el modelo conserve los pesos correspondientes a la poca con mejor desempeo en validacin. Esto evita quedarse con pesos subptimos de pocas posteriores.

6.2 Proceso de entrenamiento

El entrenamiento se realiza mediante la funcin model.fit, utilizando:

train_ds como datos de entrenamiento (incluye normalizacin y, cuando corresponde, data augmentation)

val_ds como datos de validacin (sirve para monitorear el desempeo en cada poca)

30 pocas mximas Aunque el Early Stopping suele detener antes de llegar al lmite.

Callbacks = [early_stop] Para activar el monitoreo automtico durante el entrenamiento.

Modelo 1 CNN Base

El primer modelo desarrollado fue una Red Neuronal Convolucional (CNN) base, la cual sirve como punto de referencia para comparar el desempeo de las arquitecturas ms avanzadas construidas posteriormente. Este modelo utiliza una estructura clsica y relativamente ligera, diseada para capturar patrones visuales simples y complejos presentes en los dgitos escritos a mano por estudiantes.

El modelo comienza con una serie de capas convolucionales. La primera capa aplica 32 filtros 33 con funcin de activacin ReLU y padding same, lo que permite detectar bordes y trazos sin perder resolucin espacial. Posteriormente, una capa de MaxPooling reduce la dimensionalidad, conservando la informacin ms relevante. A esta le sigue una segunda capa convolucional con 64 filtros, nuevamente acompaada de MaxPooling para profundizar la extraccin de caractersticas. Finalmente, se incluye una tercera capa convolucional con 128 filtros, cuya funcin es reconocer patrones de mayor nivel asociados a las diferencias entre dgitos.

Despus de estas etapas de extraccin de caractersticas, los mapas convolucionales se transforman en un vector mediante una capa Flatten, que conecta con una capa densa de 64 neuronas con activacin ReLU. En esta seccin se incluye tambin una capa Dropout del 30%, que ayuda a reducir el sobreajuste apagando neuronas aleatoriamente durante el entrenamiento. La ltima capa es una capa densa de 10 neuronas con activacin softmax, encargada de generar la probabilidad de pertenencia a cada una de las clases (dgitos del 0 al 9).

El modelo se entren utilizando el optimizador Adam, con la funcin de prdida categorical_crossentropy, adecuada para clasificacin multiclase. Adicionalmente, se implement Early Stopping, un mecanismo que detiene el entrenamiento cuando la exactitud de validacin deja de mejorar durante varias pocas consecutivas, evitando as el sobreentrenamiento y asegurando que el modelo conserve los mejores pesos obtenidos.

Modelo 2 CNN con Batch Normalization

El segundo modelo implementado corresponde a una versin mejorada de la arquitectura base mediante la incorporacin de Batch Normalization despus de cada capa convolucional y densa. Esta tcnica es ampliamente utilizada en modelos modernos por su capacidad para estabilizar y acelerar el proceso de entrenamiento, adems de mejorar la capacidad de generalizacin del modelo.

En su estructura inicial, el modelo mantiene las mismas tres capas convolucionales del modelo base: una primera con 32 filtros, una segunda con 64 filtros y una tercera con 128 filtros; todas con tamaos de kernel de 33, activacin ReLU y padding same. Sin embargo, tras cada una de estas capas se agrega una capa de Batch Normalization, cuyo propsito es normalizar las activaciones internas por lote. Esto reduce la sensibilidad del modelo a los cambios en la distribucin de los datos, evitando que ciertas neuronas dominen el aprendizaje y permitiendo que la red entrene de manera ms estable.

Despus del bloque convolucional, el modelo utiliza una capa Flatten seguida de una capa densa de 64 unidades con activacin ReLU. Tambin en esta etapa se incluye Batch Normalization, lo cual ayuda a que la capa completamente conectada aprenda patrones ms consistentes y evite saturacin en las activaciones. Posteriormente, se incorpora un Dropout del 30%, que reduce el riesgo de sobreajuste apagando neuronas de manera aleatoria durante el entrenamiento.

La capa final es una capa densa con 10 neuronas y activacin softmax, encargada de producir la distribucin de probabilidades entre los dgitos del 0 al 9. El modelo se compila con el optimizador Adam y la funcin de prdida categorical_crossentropy, igual que en el modelo base, para garantizar un entrenamiento eficiente en un problema multiclase.

El entrenamiento se realiza utilizando el mismo conjunto de entrenamiento y validacin, junto con la misma estrategia de Early Stopping, que detiene el aprendizaje cuando la precisin de validacin deja de mejorar. Esto permite comparar de manera directa el desempeo entre la versin con Batch Normalization y la arquitectura base.

Modelo 4 Arquitectura CNN Ligera (Light Model)

El cuarto modelo corresponde a una versin optimizada y ligera de la red convolucional, cuyo propsito es reducir significativamente el nmero de parmetros sin sacrificar en exceso el rendimiento. Para lograr esto, la arquitectura reemplaza las capas convolucionales tradicionales por Separable Convolutional Layers (SeparableConv2D), una variante eficiente que divide la operacin convolucional en dos etapas: una convolucin depthwise seguida de una convolucin pointwise. Este mecanismo permite extraer caractersticas espaciales de manera ms econmica, reduciendo el costo computacional y la memoria requerida durante el entrenamiento.

La red inicia con una capa SeparableConv2D de 32 filtros, seguida por una capa de max pooling para reducir la dimensionalidad. Posteriormente incorpora una segunda capa separable de 64 filtros, acompaada nuevamente por max pooling. Estas capas permiten que el modelo aprenda patrones simples y medianamente complejos utilizando muchos menos parmetros que un CNN tradicional, lo que es particularmente til cuando se busca rapidez de entrenamiento o se dispone de recursos computacionales limitados.

Tras la etapa convolucional, la arquitectura utiliza Flatten para convertir los mapas de caractersticas en un vector unidimensional, seguido de una capa densa de 64 neuronas con activacin ReLU. Esta capa acta como un clasificador intermedio que combina las caractersticas extradas por las capas separables. Finalmente, el modelo concluye con una capa Dense(10, softmax) que genera las probabilidades para cada una de las diez clases del problema.

El modelo se compila utilizando el optimizador Adam y la funcin de prdida categorical_crossentropy, manteniendo la consistencia con los modelos anteriores. El entrenamiento incorpora la misma estrategia de Early Stopping, lo que permite comparar su desempeo directamente contra las dems arquitecturas evaluadas.

En general, este modelo ligero ofrece una alternativa eficiente que mantiene una buena capacidad de reconocimiento, reduciendo al mismo tiempo la cantidad de parmetros y el riesgo de sobreajuste. Aunque su precisin puede ser ligeramente inferior a la de arquitecturas ms grandes, su velocidad y simplicidad lo convierten en una opcin muy prctica y equilibrada para problemas de clasificacin de dgitos manuscritos.

Modelo 5 CNN con Data Augmentation

El quinto modelo utiliza la misma arquitectura base del primer modelo, pero incorpora una mejora clave en el proceso de entrenamiento: Data Augmentation. Esta tcnica consiste en aplicar transformaciones aleatorias a las imgenes durante el entrenamiento, con el objetivo de ampliar artificialmente el conjunto de datos y aumentar la variedad de patrones disponibles para la red. Esto es especialmente til cuando las muestras son limitadas o cuando existe alta variabilidad en la forma en la que los estudiantes escriben los dgitos.

La arquitectura mantiene tres capas convolucionales con 32, 64 y 128 filtros respectivamente, todas con kernels de 33, activacin ReLU y padding 'same'. Estas capas extraen caractersticas visuales desde patrones simples como lneas verticales, horizontales y curvas bsicas, hasta representaciones ms complejas relacionadas con la forma del dgito en cuestin. Despus de la etapa convolucional, el modelo utiliza una capa Flatten, seguida de una capa densa de 64 unidades, donde se integran las caractersticas aprendidas. Tambin se incluye un Dropout del 30%, cuyo objetivo es minimizar el sobreajuste al desactivar aleatoriamente una parte de las neuronas durante el entrenamiento. Finalmente, la capa Dense(10, softmax) produce las probabilidades de las diez clases correspondientes a los dgitos del 0 al 9.

La diferencia fundamental de este modelo reside en que la red se entrena con imgenes aumentadas, generadas mediante rotaciones, zooms, traslaciones y otros cambios aleatorios aplicados a las muestras de entrenamiento. Esto permite que el modelo aprenda a generalizar mejor frente a variaciones en el trazo, tamao, inclinacin o posicin de los dgitos manuscritos. Adems, este enfoque reduce la dependencia del modelo a patrones muy especficos y mejora su capacidad para reconocer nmeros escritos con diferentes estilos.

El modelo se compila nuevamente con el optimizador Adam y la funcin de prdida categorical_crossentropy, manteniendo la consistencia metodolgica con los modelos anteriores. El entrenamiento se realiza utilizando Early Stopping para detener el aprendizaje cuando la precisin en validacin deja de mejorar.

En general, este modelo busca mejorar la robustez y la capacidad de generalizacin sin modificar la arquitectura original. Aunque su precisin puede ser comparable o ligeramente menor segn el comportamiento del dataset real, su principal ventaja radica en que reconoce mejor dgitos que presentan variaciones atpicas, ruido o pequeas deformaciones, lo cual es esencial en un escenario de escritura humana.
